Const修饰指针

C语言可以通过修改指针的方式修改const变量，而C++不可以，会报错

C++中的const修饰的变量认为是常数，即在编译时，使用const变量的位置直接替换成常数

C++中的const修饰的变量可以作为数组长度，而C不可以

const修饰指针的情况：能力可缩小，不可放大

能力强的指针可以赋值给能力小的指针，高权限的资源，交给低权限的指针去管理 —— 编译器认为这是安全的（因为低权限指针不会做超出原资源权限的操作）。

高能力：int*能修改指向的值，能修改指针指向

中能力：const int*只能读指向的值，能修改指针朝向，但不能修改指向的值

低能力：const int* const不能修改指向的值，不能修改指针朝向

核心是编译器的安全校验逻辑：
能力缩小（合法）：
比如把 int* 赋值给 const int*，相当于「本来能随便改的资源，现在只允许读」—— 不会破坏原资源的约束，编译器允许。
能力放大（非法）：
比如把 const int* 赋值给 int*，相当于「本来只能读的常量资源，现在允许通过指针修改」—— 这违背了 const 定义的初衷（常量不可修改），编译器直接报错拦截。

**!**指针变量名之前的const，赋值时不考虑能力是否缩小

例如，int*const p在参与赋值时，可以认为其是高能力指针

引用

基本介绍

语法：数据类型&引用名=变量名

引用名是变量的别名，是变量的本身

引用的特点：必须初始化值，不能为NULL，不可更改位置，没有独立的内存空间

引用的用法同变量

引用没有二级引用

指针的引用

指针变量的引用：数据类型*&引用名=指针变量

数组的引用

数组的别名：数据类型(&别名)[长度]=数组名

引用的本质

引用的本质是在编译时将引用优化为了指针

普通引用优化成了指针常量:int*const

const引用

const引用的特点是“可以绑定各种值，但不能通过它修改原值”

const引用绑定常数时，译器会先创建一个临时的int对象存储 1，再让 const 引用绑定这个临时对象，还会延长这个临时对象的生命周期（和引用同寿命）

引用和指针作为函数形参的区别

指针：是一个独立的变量，存储的是另一个变量的内存地址。可以为空（nullptr），也可以指向不同的地址。

引用：是一个变量的别名，不是独立变量。必须在定义时初始化，且一旦绑定某个变量，就不能再指向其他变量，也不能为空。不占用额外内存空间

用引用的场景：

希望函数修改传入的变量，且能保证传入的变量非空；

追求语法简洁（无需解引用）；

实现运算符重载（如 operator[]、operator=），提升代码可读性。

用指针的场景：

需要允许传入空值（比如可选参数）；

需要动态改变指向的对象（比如链表节点的指针重定向）；

内联函数(inline)

内联函数是编程语言中的一种函数定义方式，它建议编译器在调用该函数的地方将函数体直接展开，而不是进行常规的函数调用（压栈、跳转、返回等）。其目的是减少函数调用的开销，提高程序的执行效率。

工作原理

当编译器看到一个函数被声明为 inline 时，它会在每个调用该函数的地方，用函数体的代码替换函数调用指令。

使用场景

函数体非常小（如只有一两行代码），且被频繁调用（例如循环中的简单计算）。

避免宏的副作用（宏是文本替换，内联函数有类型检查、作用域等）。

优点

消除调用开销：省去参数压栈、跳转、返回等指令，适合高频调用的小函数。

优化机会更多：展开后编译器可对函数体与周围代码进行整体优化（如常量传播）。

保留类型安全：比宏更安全，支持重载、作用域和语法检查。

缺点

代码膨胀：如果函数体较大或调用点过多，会导致可执行文件体积增大，可能反而降低指令缓存命中率，影响性能。

编译时间增加：每次修改内联函数，所有调用它的文件都需要重新编译。

非强制：inline 只是对编译器的建议，编译器可能忽略（例如递归函数、包含复杂控制流、函数地址被取用时）。

与宏的区别

宏：预处理器直接文本替换，无类型检查，可能产生副作用（如 SQUARE(x++) 展开两次）。

内联函数：编译器处理，有类型和作用域，行为与普通函数一致，只是调用方式被优化。

缺省参数

缺省参数（也叫默认参数）是指在定义函数时，为某些参数预先指定一个默认值。如果在调用函数时没有传入这些参数的值，函数就会自动使用这个默认值，从而简化调用方式。

主要特点

让函数调用更灵活，可以省略某些参数。

避免因参数缺失而报错。

语法与规则

缺省参数在函数声明或定义时通过 = 赋值指定。

必须从右向左连续缺省：一旦某个参数设定了默认值，它右边所有参数都必须有默认值。

通常只在函数声明中给出缺省值，定义中不再重复（若重复则需保持一致）。

缺省值可以是常量、全局变量、函数调用等，但必须在编译时可确定。

注意事项

位置限制：缺省参数必须在参数列表的末尾

声明与定义：默认值通常只在声明中提供，定义中不再写默认值(除非函数没有单独声明)

不可跳过：调用时实参按从左到右匹配，不能跳过前面的缺省参数去指定后面的

函数重载

函数重载是编程语言中的一种特性，允许在同一个作用域内定义多个同名的函数，但这些函数的参数列表必须不同（即参数的个数、类型或顺序有所区别）。编译器或解释器会根据调用时传入的实参类型和个数，自动选择匹配的函数版本进行调用。

主要特点

函数名相同：便于记忆和使用，提高代码可读性。

参数列表不同：这是区分重载函数的唯一依据。参数的不同可以体现在：

参数的类型不同

参数的个数不同

参数的顺序不同（只要类型不同）

返回类型无关：仅返回类型不同不足以构成重载，因为调用时无法仅凭返回类型确定调用哪个函数。

作用域相同：重载的函数必须在同一个类或同一个命名空间内。

优点和注意事项

优点

提高代码的清晰度和一致性（同一功能使用同一函数名）。

方便调用者，无需记忆多个不同名的相似函数。

增强代码的扩展性，便于添加新的参数类型。

注意事项

重载决议发生在编译期，根据静态类型确定调用版本。

若调用时实参无法精确匹配任何一个重载版本，编译器会尝试隐式类型转换，可能导致歧义错误。

并非所有语言都支持重载，需根据语言特性合理设计。

缺省不构成函数重载

基本概念

缺省值（默认参数）：在函数声明时为参数指定一个默认值，调用时如果省略该参数，则使用默认值

函数重载：定义多个同名函数，但参数列表不同（参数个数、类型或顺序不同）。调用时根据实参匹配对应的版本。

为何不构成？

本质区别：缺省值是在同一个函数上提供灵活性，而重载是定义多个不同的函数。

编译后符号：带有默认参数的函数在编译后仍然是一个函数符号；重载的函数则生成多个不同修饰名的符号。

调用决议：默认参数由编译器在调用处补齐，属于编译时行为；重载则需要根据实参选择最匹配的函数。

函数重载匹配顺序

函数重载匹配的本质是编译器按照 “最优匹配” 原则，从所有候选函数中筛选出最合适的一个，整个过程分为三个优先级阶段，编译器会按顺序检查，找到第一个匹配成功的函数就停止，若同一阶段找到多个匹配则报错（二义性）

匹配顺序（优先级从高到低）

1. 精确匹配

这是最高优先级，编译器首先寻找 “完全一致” 的函数，无需任何类型转换。

2.提升匹配

若没有精确匹配，编译器会寻找 “类型提升后匹配” 的函数，提升是无精度损失的基础类型转换。

常见的类型提升：

char/unsigned char/short → int；

unsigned short → int（若int能容纳其值，否则到unsigned int）；

float → double；

枚举类型 → int。

3.标准转换匹配

若前两级都无匹配，编译器会寻找 “标准类型转换后匹配” 的函数，这类转换可能有精度损失，优先级最低。

常见的标准转换：

数值类型转换（如int→double、double→int、int→unsigned int）；

指针转换（如void*→int*、派生类指针→基类指针）；

算术类型与布尔类型的转换（如int 0→bool false，int 5→bool true）。

Const修饰的变量能否构成函数重载？

const修饰的变量不构成函数重载，但是const修饰的引用和指针可以构成函数重载

核心原理：函数签名的判定标准

函数重载的关键是「编译器能否通过参数的本质差异区分不同函数」：

const 修饰普通变量（如 int）：仅限制变量本身不可修改，参数的类型和传递方式无变化，因此不改变函数签名；

const 修饰指针 / 引用：改变了「指针 / 引用指向的对象是否可修改」，本质是参数类型发生了变化，因此会改变函数签名，构成重载。

const修饰普通变量

普通变量作为函数参数时，const 仅作用于函数内部的参数副本，对函数签名无影响 —— 编译器认为这是同一个函数

原因：int a 和 const int a 传递的都是「值」，参数类型都是 int，const 只是限制函数内部不能修改 a，但对函数签名无影响

const修饰指针

const 修饰指针有两种情况：
const int* p：指针指向的对象不可修改（指向常量的指针）；
int* const p：指针本身不可修改（指针常量）—— 注意：这种情况不构成重载（因为指针本身的 const 不影响传参时的类型匹配），下文重点讲第一种（核心场景）

const修饰引用

引用的本质是 “变量的别名”，const 修饰引用时，限制了 “通过引用修改原变量”，参数类型发生变化，因此构成重载。

函数调用约定

函数调用约定（Calling Convention）是编译器规定的一套规则，用来决定：

  1. 函数参数怎么传递（栈 / 寄存器）
  2. 参数入栈顺序
  3. 调用结束后谁来清理栈（调用者 / 被调用者）
  4. 函数名编译后的修饰规则（C++ 特有）

它是调用方和被调用函数之间的契约，不遵守就会栈崩溃、程序报错。

最常用的四种调用约定

1._cdecl(C/C++默认)

参数入栈：从右向左

栈清理：调用者清理（caller cleanup）

特点：支持可变参数（printf、scanf）

名字修饰：C 语言加下划线 _func；C++ 是复杂修饰名

优点：通用、支持可变参

缺点：每个调用点都要生成清栈代码，体积略大

2._stdcall(Windows API默认)

参数入栈：从右向左

栈清理：被调用者清理（callee cleanup）

特点：不支持可变参数

名字修饰：_func@数字（数字 = 参数总字节数）

优点：代码体积小、效率高

缺点：不支持可变参

_fastcall(快速调用)

用途：追求速度，用寄存器传参

参数：前 2 个参数用 ECX、EDX 寄存器，剩下的栈传参

入栈：从右向左

清栈：被调用者清理

_thiscall(C++类成员函数专用)

__thiscall 是 C++ 独有的调用约定，专门用于非静态类成员函数

核心规则：

this 指针传递

this 指针直接通过 ECX 寄存器 传递（核心特征）

参数入栈顺序：从右向左

栈清理：

无可变参 → 被调用者清理栈

不支持可变参数

名称空间

为何需要名称空间

命名冲突问题：大型项目中不同模块可能使用相同的标识符（函数、类、变量等）

代码组织：将逻辑相关的实体归类到同一作用域下

标准库的隔离：所有标准库组件都在 std 名称空间中

基本用法

定义名称空间

名称空间可以定义在全局作用域、其他名称空间内、甚至函数内（但不能在块作用域内）

名称空间是开放的：可以在多个文件中多次定义同一个名称空间，内容会合并

使用名称空间中的成员

作用域解析运算符 ::

全局名称空间

全局作用域本质上是一个隐式的名称空间，可以用 :: 显式访问

嵌套名称空间

可用 using namespace Outer::Inner; 简化访问

常用于库的版本管理

标准库名称空间

所有标准库组件位于 std 中

用户自定义类型可特化 std 中的模板（如 std::hash），但必须遵循规定

右值引用

左值与右值

左值（lvalue）：有名字、可取地址的表达式，通常能出现在赋值号左边。

如变量、返回左值引用的函数调用等。

右值（rvalue）：临时对象、字面量等，没有持久地址，通常出现在赋值号右边。

C++11 引入右值引用的动机

传统左值引用（T&）只能绑定左值，无法区分临时对象。

临时对象（右值）的资源常常可以被“窃取”而无需深拷贝，从而提高性能。

！右值引用本身是左值

一旦右值引用被命名，它在作用域内就变成左值（因为它有名字、可取地址）

右值引用的语法

右值引用用 && 声明：T&&

可以绑定到右值，但不能直接绑定左值（除非使用 std::move 强制转换）。

右值引用可以延长临时对象的生命周期（类似于 const 左值引用）

右值引用构成函数重载

重载规则

可以基于参数是左值引用还是右值引用进行重载，从而区分处理左值和右值

重载解析的要点

左值优先匹配左值引用版本，右值优先匹配右值引用版本。

如果只提供右值引用版本而不提供左值引用版本，则左值无法直接调用（需用 std::move 转换）。

内存动态管理

回顾C的动态内存

C中内存动态管理的函数：malloc，realloc，calloc，free

C++的动态内存

C++中内存动态管理有三种方式

new和delete(常用方式)

**特点：**自动计算大小，可以初始化，失败则抛出异常

operator new和operator delete(底层分配)

只分配空间，不调用构造函数，等同于 malloc。

通常与“定位new”配合使用，手动构造对象。

定位new

在已分配的内存上构造对象。

常与 operator new 或自定义内存池配合。

decltype

decltype 是 C++11 引入的关键字，它用来获取一个表达式的类型，而且不会计算该表达式。简单说：你给它一个变量或表达式，它告诉你这个变量或表达式的准确类型（包括 const、引用、指针等一切修饰）。

基本用法

对变量名使用 decltype，得到的就是该变量声明时的类型（不会去掉 const 或引用）。

如果给的不是变量名，而是一个表达式，那么得到的是该表达式结果的类型。

关键规则

变量名 → 直接得到它的声明类型。

表达式（例如函数调用、算术运算、解引用等）→ 得到表达式结果值的类型，而且会保留引用等属性。

加上括号的变量名（如 (x)）会被当作表达式，规则与表达式一致。

decltype可用来声明函数

decltype(hi) 推导出的类型就是 hi 的函数类型，即 void(const char*)。

因此 decltype(hi) func; 等价于 声明一个函数 func，它的参数列表和返回类型与 hi 完全相同。

decltype可用来声明并初始化函数指针

&hi 是取 hi 的地址，它的类型是函数指针类型 void(*)(const char*)。

decltype(&hi) 就得到这个函数指针类型。

func2 是一个函数指针变量，用 hello 函数（同样为 void(const char*) 类型）来初始化。

之后可以通过 func2("老杨"); 调用 hello 函数

nullptr

在nullptr之前，空指针通常用 NULL 宏表示。NULL 通常被定义为 0 或 (void*)0。这导致两个问题：

二义性：NULL 既是整数 0，又表示空指针，在重载时可能产生歧义。

类型不安全：将 NULL 传递给期望整数参数的函数时，可能意外匹配到指针版本，或相反。

nullptr的特性

nullptr 是一个关键字（C++11 起），表示空指针常量。

它的类型是 std::nullptr_t（定义在 <cstddef> 中）。

nullptr 可以隐式转换为任何指针类型或成员指针类型，但不能隐式转换为整数类型。

这消除了与整数 0 的歧义，使重载解析更清晰。

使用场景

初始化指针：统一使用 nullptr 代替 0 或 NULL。

重载函数区分：使指针版本与整数版本明确区分。
