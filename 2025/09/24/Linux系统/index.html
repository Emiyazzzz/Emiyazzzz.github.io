预编译阶段

删除所有的注释;
添加行号和文件名标识，以便于编译器产生调试用的符号信息及编译时产生编译错

进程

进程概念

进程是计算机中正在运行的程序，每一个进程都由操作系统进行管理，并且有自己的内存空间，代码，数据，堆栈等

进程状态：运行，就绪，阻塞，终止

进程标识符

PID：每个进程都有一个唯一的标识符，称为进程ID，操作系统通过PID来管理进程

PPID：父进程的PID，用来表示进程之间的父子关系

进程控制块

PCB 是操作系统用来存储进程信息的数据结构，包括进程的状态、程序计数器、CPU 寄存器、内存管理信息等。操作系统通过 PCB 来管理和调度进程。

PCB 包含的信息：

·进程状态

·程序计数器

·CPU 寄存器

·内存管理信息

·文件管理信息

·进程的 PID 和 PPID

进程创建与终止

创建进程

**·**使用fork()系统调用创建子进程。父进程调用fork()会返回子进程的PID，子进程则接受到一个返回值为0

**·**fork()后父进程和子进程会有几乎相同的地址空间，但他们是独立的进程

终止进程

**·**进程结束时，操作系统会通过exit()系统调用终止进程并清理资源。父进程可以通过wait()或waitpid()等待子进程结束

进程调度与优先级

调度算法：操作系统使用不同的算法来调度进程执行，如轮转调度（Round Robin）、优先级调度、最短作业优先等。

优先级：进程的优先级决定了它能被调度的优先级，高优先级的进程会先获得 CPU 时间。

进程间通信(IPC)

进程间通信是不同进程之间交换数据的一种机制。常见的IPC机制有：

**管道：**提供进程间单向数据流。

**消息队列：**通过队列传递消息。

**共享内存：**多个进程共享一块内存区域。

**信号：**用于通知进程发生某些事件（如中断、终止等）。

**套接字：**用于不同计算机间的进程通信。

信号与进程控制

信号

信号是一种进程间的异步通信机制，操作系统通过信号通知进程发生了某些事件。例如，SIGINT表示中断信号，SIGKILL表示终止进程。

父子进程

父进程创建子进程之后，可以使用wait()或waitpid()等待子进程结束，获取子进程的退出状态

如果父进程在子进程结束之前终止，操作系统会将子进程的父进程 ID 改为 1，成为孤儿进程，由 init 进程管理。

进程调度与上下文切换

上下文切换：当操作系统切换当前运行的进程时，需要保存当前进程的状态（如寄存器、程序计数器等），并加载下一个进程的状态。这一过程称为上下文切换。

**调度器：**操作系统的调度器负责决定哪个进程能在某一时刻获得 CPU 时间。

其他问题

fork与exec的区别

**fork()**创建一个新的进程，子进程是父进程的副本

**exec()**用另一个程序替换当前进程的代码和数据，执行不同的程序

僵死进程与避免方法

原因：子进程已经退出，但父进程没有回收他的退出状态，导致PCB残留

避免方法：父进程等待子进程退出，拿走退出状态，内核就会清理子进程

文件描述符与系统调用

文件描述符

文件描述符（FD）：文件描述符是一个整数，操作系统用它来标识一个打开的文件。每个进程都有自己的文件描述符表。

标准输入（stdin）：文件描述符 0。

标准输出（stdout）：文件描述符 1。

标准错误（stderr）：文件描述符 2。

文件描述符的作用：

文件描述符用于与文件或设备进行交互。当你在程序中打开文件时，操作系统返回一个文件描述符，它作为你后续操作文件的引用。

操作系统通过文件描述符来管理打开的文件，包括读取、写入、关闭等操作

系统调用与文件操作

系统调用是用户程序与操作系统内核之间的接口，常见的文件相关系统调用包括：

打开文件：open()

open() 系统调用用于打开文件并返回文件描述符。文件可以是普通文件、目录、设备文件等。

open() 的常用标志：

O_RDONLY：只读模式打开文件。

O_WRONLY：只写模式打开文件。

O_RDWR：读写模式打开文件。

O_CREAT：如果文件不存在则创建。

O_APPEND：追加模式。

读取文件：read()

read() 系统调用用于从打开的文件中读取数据。

参数：

fd：文件描述符。

buffer：存储读取数据的缓冲区。

count：要读取的字节数。

写入文件：write()

write() 系统调用用于向文件中写入数据。

参数：

fd：文件描述符。

buffer：包含要写入的数据的缓冲区。

count：要写入的字节数。

关闭文件：close()

close() 系统调用用于关闭文件描述符，释放与文件相关的资源。

文件定位：lseek()

lseek() 系统调用用于设置文件指针的位置，常用于随机访问文件。

SEEK_SET：从文件开头设置偏移量。

SEEK_CUR：从当前文件指针设置偏移量。

SEEK_END：从文件末尾设置偏移量。

文件描述符表

每个进程都有一个 文件描述符表，该表存储了进程打开的所有文件的文件描述符及其对应的文件信息（如文件偏移量、访问权限等）。

文件描述符表中，文件描述符通常从 0 开始递增，依次为标准输入、标准输出、标准错误以及其他打开的文件。

共享文件描述符：

当一个进程调用 fork() 创建子进程时，父进程的文件描述符表会被子进程继承。父子进程可以共享相同的文件描述符。

其他问题

文件描述符是什么

文件描述符 是操作系统为每个打开的文件分配的一个整数值，用来标识文件。每个进程有自己的文件描述符表，用于管理打开的文件。

如何创建和关闭文件描述符

创建文件描述符：使用 open() 系统调用。

关闭文件描述符：使用 close() 系统调用。

read() 和 write()系统调用是如何工作的？

read(fd, buffer, count)：从指定文件描述符 fd 读取最多 count 字节的数据，存储到 buffer 中。

write(fd, buffer, count)：将 buffer 中最多 count 字节的数据写入文件描述符 fd 指向的文件中。

文件描述符的重定向是什么？

重定向是将标准输入、输出或错误流重定向到文件或其他设备。可以通过 shell 命令或系统调用（如 dup()、dup2()）来实现。

dup() 和 dup2() 函数的作用是什么？

dup()：复制一个文件描述符，返回新的文件描述符，指向相同的文件。

dup2()：复制一个文件描述符到指定的文件描述符，如果原文件描述符已经打开，先关闭它。

fork() 后，父子进程如何共享文件描述符？

父子进程通过 fork() 创建后，共享同一文件描述符表。这意味着父进程和子进程可以同时访问相同的文件。

内存资源管理

内存管理基本概念

内存管理是操作系统的一个核心功能，它负责分配、使用和回收计算机内存资源，确保程序可以有效地使用内存，同时保证系统的稳定性和性能。

**内存分配：**操作系统根据需求为进程分配内存。

内存释放：当进程完成任务或终止时，操作系统需要回收分配给该进程的内存。

**内存保护：**通过防止进程访问未分配或非法的内存区域，确保进程之间不会相互干扰。

**内存共享：**操作系统支持进程间共享内存，以便它们可以共享数据。

内存管理方式

操作系统使用不同的策略来管理内存：

**单一连续分配：**将整个内存作为一个连续的区域分配给程序，这种方式简单但容易导致内存碎片问题。

分区管理：

将内存分割成多个固定大小或动态大小的块，分配给不同的进程。

分区可以是 固定分区 或 动态分区。

分页管理：

将内存和程序分成固定大小的 页面 和 页框。一个页面大小为4k

通过分页机制，操作系统可以将不连续的物理内存分配给进程的虚拟内存地址空间，从而消除了内存碎片问题。

分段管理：

将内存分为多个不等长的段，每个段代表程序的一个逻辑部分（如代码段、数据段、堆栈段等）。

分段管理允许更灵活的内存分配和共享。

虚拟内存：

通过 虚拟内存，操作系统可以允许程序使用比实际物理内存更多的内存。虚拟内存使用硬盘（如交换空间或交换文件）作为物理内存的扩展，避免程序因内存不足而崩溃。

内存访问控制与保护

**内存保护：**通过操作系统的硬件支持，确保进程只能访问其分配的内存区域。通常通过硬件的页表和保护位实现。

保护位可以设置为只读或可写，防止非法访问。

虚拟地址与物理地址映射：操作系统通过 页表 将程序使用的虚拟地址映射到物理地址。每个进程有独立的虚拟地址空间，操作系统负责管理虚拟地址与物理内存之间的映射关系。

内存映射与共享内存

内存映射：操作系统允许进程通过 内存映射文件 或 设备文件，直接在内存中映射外部文件，从而避免文件 I/O 操作带来的性能损失。

普通读文件：
·打开文件
·内核把文件内容读到内核缓冲区
·再复制到用户空间缓冲区
·才能用
内存映射：
·直接把文件映射到你进程的地址空间
·读写这块 “内存” = 直接读写文件
·少一次拷贝，更快、更省内存

内存映射的作用

·大文件高效读写

·进程间共享内存：多个进程映射同一个文件=天然共享内存

·动态库加载：Linux/WIndows的.so/.dll都是用内存映射加载到

·减少数据拷贝，提高IO性能

共享内存

多个进程可以通过共享内存来交换数据。这是 IPC（进程间通信）中一种非常高效的方式。

内存碎片

内存碎片指的是内存空间的浪费，分为两种类型：

**外部碎片：**内存中有足够的总空间，但由于内存被分割成不连续的块，导致无法分配给大块的进程。

**内部碎片：**为满足分配请求，操作系统分配的内存块大小大于实际需求，导致剩余部分不能被利用。

内存碎片问题的解决办法

使用 分页管理：将内存分成固定大小的页面，避免外部碎片。

使用 内存合并：当一个空闲内存块的两边是空闲的，可以将其合并，减少碎片。

使用 最佳适配：分配内存时选择最小的适合块，以避免浪费空间。

写时拷贝（COW）

写时拷贝的概念

写时拷贝是一种延迟内存复制的技术。当多个进程共享内存时，内存的复制不会立即发生，只有在某个进程尝试修改共享内存时，才会发生复制。

写时拷贝是如何工作的

在进程通过 fork() 创建时，父子进程共享同一块内存。直到某个进程修改内存时，操作系统才会复制该内存页，这种复制操作称为 写时拷贝。

为何使用写时拷贝

写时拷贝通过延迟内存复制，减少了不必要的内存开销，提高了性能，并且节省了内存资源。它特别适用于需要创建大量进程的场景。

写时拷贝的局限性

频繁的写操作可能导致内存复制的开销较大，从而影响性能。此外，写时拷贝增加了操作系统内存管理的复杂性。

写时拷贝如何提高内存管理的效率

写时拷贝允许多个进程共享相同的内存空间，直到需要修改内存时才进行复制，这样避免了不必要的内存分配和复制，从而提高了内存利用效率。

信号

信号 是操作系统用于通知进程发生某些事件的机制。它是进程间异步通信的一种方式，用于通知一个进程某个事件的发生（例如，中断、异常、定时器事件等）。通常用于进程间通信、异常处理和中断处理。

常见的信号包括：

  • SIGINT：中断信号（Ctrl+C）。
  • SIGTERM：终止信号。
  • SIGKILL：强制终止信号。
  • SIGSEGV：段错误信号。
  • SIGALRM：定时器信号。
  • SIGCHLD：子进程结束信号。
  • 用户自定义信号如 SIGUSR1 和 SIGUSR2

发送信号

使用 kill(pid, sig) 函数向进程发送信号。其中 pid 是进程 ID，sig 是信号类型。

接受信号并处理

进程可以通过 signal() 或 sigaction() 设置自定义的信号处理函数。

signal() 示例：

signal() 是一个简化的接口，用于处理信号。

sigaction() 提供了更强大的功能，允许对信号的处理做更多配置，比如指定信号处理函数的行为。

信号队列 是指操作系统在信号处理之前将其排队。进程可以按照顺序处理信号，避免遗漏信号。

线程

线程的基本概念

线程是操作系统能够独立调度的最小单位。它是进程中的一个执行流，进程可以包含一个或多个线程。线程是计算机中的基本调度单位，一个进程可以有多个线程，它们共享进程的资源（如内存空间、文件句柄等），但每个线程有独立的栈、寄存器和程序计数器。

**进程：**一个执行中的程序，包含了代码、数据、堆栈和堆等。

**线程：**进程内的执行单位，共享进程资源，独立执行。

线程的创建

在多线程编程中，线程的创建通常使用操作系统提供的线程库或 API。

C 语言线程创建（pthread）：使用 POSIX 线程（pthread）库来创建和管理线程。

pthread_create()：创建新线程。

pthread_join()：等待线程结束。

pthread_exit()：退出线程。

并发与并行

**并发：**指的是多个线程在同一时段内交替执行，虽然它们可能不是同时执行的，但操作系统通过时间片轮转等技术实现了并发的效果。

**并行：**指的是多个线程在多核处理器上同时执行，每个线程在不同的核心上独立运行，真正意义上的同时执行。

线程的同步

多线程程序中的多个线程可能会同时访问共享资源，导致数据竞争和不一致的问题。为了避免这些问题，必须使用线程同步机制来控制线程间的执行顺序和资源访问。

**互斥锁（Mutex）：**用于确保在同一时刻只有一个线程可以访问共享资源。

**信号量：**用于控制多个线程对共享资源的访问，通过计数器来管理资源的使用。

**条件变量：**用于线程间的协调，使得线程在满足某些条件时才继续执行。

读写锁

线程的调度

操作系统的调度程序决定线程何时开始执行、何时挂起以及何时恢复。调度策略可以是：

抢占式调度：操作系统可以中断正在执行的线程，切换到另一个线程。

非抢占式调度：线程在执行时不会被中断，直到它主动放弃 CPU 使用权。

死锁

死锁 是指多个线程因相互等待对方释放资源而导致的无限等待。避免死锁的方法包括：

按照相同的顺序获取多个锁。

使用锁超时机制。

避免循环等待。

线程安全

线程安全是指多个线程并发访问某个函数、数据结构或对象时，不会出现数据竞争（race condition），且不会导致程序的行为不一致或崩溃。换句话说，线程安全的代码可以保证在多线程环境下，多个线程同时执行时，不会产生错误的结果。

保证线程安全的方法

1.互斥锁

**2.原子操作：**原子操作是指不可分割的操作，一旦开始执行就不会被中断，保证操作的完整性。例如，增加一个计数器的操作可以通过原子操作实现，这样即使多个线程同时执行，也不会导致数据竞争。

示例：atomic_add(), atomic_compare_and_swap() 等。

3.信号量

4.条件变量

5.读写锁：读写锁允许多个线程同时读取共享资源，但在写线程访问共享资源时，其他线程（无论是读线程还是写线程）都无法访问。
