动态内存

一：堆和栈

1.栈区 (Stack)

存储内容：

函数调用时的参数、局部变量、函数返回地址等。

管理方式：

由编译器自动管理。当函数被调用时，其所需内存会在栈上自动分配（“压栈”）；当函数执行完毕返回时，这些内存会被自动回收（“出栈”）

特点：

速度快：分配和回收只是移动栈指针，效率高。

空间小：栈的大小是有限的（通常几MB），如果递归太深或局部数组太大，会导致 栈溢出 (Stack Overflow)。

生命周期：变量的生命周期与函数相同，函数结束，栈内存被回收，变量失效。

2.堆区 (Heap)

存储内容：

动态分配的内存。

管理方式：

由程序员手动控制。使用 malloc(), calloc(), realloc() 函数申请内存，使用 free() 函数释放内存。如果忘记释放，就会造成内存泄漏 (Memory Leak)。

特点：

速度慢：分配和回收需要寻找合适的内存块，管理更复杂，效率低于栈。

空间大：可用内存通常只受限于计算机的虚拟内存大小。

生命周期：从调用 malloc() 开始，到调用 free() 结束。生命周期完全由程序员控制。

3.静态区/全局区 (Static/Global)

存储内容：

全局变量 (Global Variables)

静态变量 (Static Variables)（包括静态局部变量和静态全局变量）

常量（如字符串常量 “Hello”）

管理方式：

在程序编译时就已经分配好内存，在整个程序运行期间都存在，程序结束时由操作系统回收。

特点：

生命周期长：与程序的生命周期相同。

默认初始化为零：如果没有显式初始化，编译器会自动将其初始化为0或NULL。

线程安全：需要小心多线程环境下的并发访问问题。

比喻：

栈区 (Stack)：就像酒店的客房。

自动分配和清理：客人（函数）入住时自动分配房间（内存），退房时自动清理，方便快捷。

生命周期固定：客人住多久是明确的（函数开始到结束）。

空间有限：酒店房间数量是固定的，不能无限占用。

堆区 (Heap)：就像公寓楼旁的一大片空地。

手动申请和释放：你需要向管理员（操作系统）申请一块地皮（内存）来盖自己的房子，并且用完必须自己拆掉（释放），否则就永远占着那块地（内存泄漏）。

生命周期灵活：房子想盖多久就盖多久，直到你主动拆除。

空间巨大：可用的空地通常比酒店房间多得多。

静态区/全局区 (Static/Global)：就像公寓楼的永久住户名录牌。

永久存在：一旦把名字写上去（定义全局变量），在整个公寓楼的存在期间（程序运行期）它都在那里。

编译时确定：名录牌的大小和名字在制作时（编译时）就固定了。

二：内存泄漏

1.定义：

内存泄漏是指程序中已动态分配的堆内存由于某种原因程序未释放或无法释放，造成系统内存的浪费，导致程序运行速度减慢甚至系统崩溃等严重后果。

2.后果：

性能下降：可用内存减少，系统需要更频繁地进行内存交换

程序崩溃：当系统无法分配更多内存时，程序会突然终止

系统不稳定：在长时间运行的服务中（如服务器），小泄漏会累积成大问题

比喻：租房子不退房

想象一下，内存就像一栋公寓楼里的房间：

申请内存 (malloc) = 向物业租一个房间

使用内存 = 你在房间里存放东西

释放内存 (free) = 退房并把钥匙还给物业

内存泄漏就像你租了房间后，一直不退房也不使用它。物业认为房间已被占用，无法再租给其他人，但事实上这个房间是空着的——这就是资源的浪费。

三：栈溢出

1.定义：

栈溢出是指程序向栈中写入数据时超出了栈空间的边界，或者递归/函数调用层次太深，导致栈空间被耗尽的情况。

2.原因：

无限递归或递归过深
过大的局部变量
缓冲区溢出

3.后果

程序崩溃：最常见的后果是程序突然终止
安全漏洞：黑客可以利用栈溢出来执行恶意代码（缓冲区溢出攻击）
数据损坏：溢出可能会覆盖其他变量的值，导致不可预测的行为

比喻：栈内存就像一个叠盘子的游戏：

每个函数调用 = 在栈顶放一个新盘子

函数返回 = 从栈顶拿走一个盘子

栈空间 = 桌子的高度是有限的

栈溢出就像你不停地往上面叠盘子，直到盘子堆得比桌子还高，最终整个盘子塔倒塌——这就是栈溢出。

四：动态内存分配

1.malloc函数

定义：

malloc 是 C 语言中用于动态内存分配的核心函数，它允许程序在运行时从堆区申请指定大小的内存空间。
基本用法：

基本用法：

#include<stdio.h>
#include<stdlib.h>
int main() {
//申请能够存放10个整数的内存空间
int* arr = (int*)malloc(10 * sizeof(int));

free(arr);//必须释放内存

}

释放之后将指针设为空指针：arr=NULL；

关键特点：

手动管理：必须显式调用 free() 释放内存
堆内存：分配的内存位于堆区，生命周期由程序员控制
不初始化：分配的内存内容是未初始化的（包含随机值）
返回 void*：需要类型转换到具体指针类型

示例：动态数组

#include<stdio.h>

#include<stdlib.h>

int main() {

int n;

printf("请输入数组大小：");

scanf_s("%d", &n);

//动态创建数组

double* scoress = (double*)malloc(n * sizeof(double));

2.calloc函数

定义：

calloc 是 C 语言中用于动态内存分配并初始化的函数，它与 malloc 类似但有一个重要区别：calloc 会将分配的内存自动初始化为零。

void* calloc(size_t num, size_t size);

参数说明：

num：需要分配的元素数量
size：每个元素的大小（字节数）
返回值：成功时返回指向分配内存起始地址的 void* 指针，失败时返回 NULL

#include<stdio.h>

#include<stdlib.h>

int main() {

int* arr = (int*)calloc(10,sizeof(int));

if (arr == NULL) {

printf("内存不足");

return 1;

​ }

//打印初始化的值（全为0）

for (int i = 0; i < 10; i++) {

printf("arr[%d]=%d\n", i, arr[i]);

​ }

free(arr);

return 0;

}

特点：

自动初始化：分配的内存内容全部初始化为0（对于指针是NULL，对于浮点数是0.0）
两个参数：分别指定元素数量和每个元素的大小
堆内存：分配的内存位于堆区，生命周期由程序员控制

3.realloc函数

定义：

realloc 是 C 语言中用于重新分配内存大小的函数，它可以在不丢失原有数据的情况下调整之前分配的内存块的大小。
函数原型：

\#include <stdlib.h>

void* realloc(void* ptr, size_t new_size);

参数说明

ptr：指向之前分配的内存块的指针（可以是 NULL）

new_size：新的内存大小（字节数）

返回值：成功时返回指向重新分配内存的指针，失败时返回 NULL

用法示例

\#include <stdio.h>

\#include <stdlib.h>

int main() {

// 初始分配5个整数的空间

int *arr = (int*)malloc(5 * sizeof(int));

if (arr == NULL) {

​ printf("初始内存分配失败！\n");

​ return 1;

}

// 初始化数据

for (int i = 0; i < 5; i++) {

​ arr[i] = i + 1;

}

// 扩展内存到10个整数

int *new_arr = (int*)realloc(arr, 10 * sizeof(int));

if (new_arr == NULL) {

​ printf("内存重新分配失败！\n");

​ free(arr); // 释放原有内存

​ return 1;

}

arr = new_arr; // 使用新的指针

// 初始化新增的空间

for (int i = 5; i < 10; i++) {

​ arr[i] = i + 1;

}

// 打印所有数据

for (int i = 0; i < 10; i++) {

​ printf("arr[%d] = %d\n", i, arr[i]);

}

free(arr); // 释放内存

return 0;

}

关键特点

调整大小：可以扩大或缩小已分配的内存块

数据保留：尽可能保留原有数据内容

可能移动：如果需要移动内存块，会自动拷贝数据

用途：扩大内存块（增加容量）缩小内存块（减少内存使用）

特殊情况：

// realloc的特殊行为

int *ptr = NULL;

// 等同于 malloc(100)

ptr = (int*)realloc(NULL, 100 * sizeof(int));

// 等同于 free(ptr)

ptr = (int*)realloc(ptr, 0);
